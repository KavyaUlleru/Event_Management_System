Step-by-Step Testing Guide

Step 1: Test User Registration

  1. Click "Sign Up" on the homepage
  2. Fill in the registration form:
    • Name: Test Participant
    • Email: participant@test.com
    • Password: test123
    • Role: Participant
  3. Click "Sign Up" - you should be redirected to the dashboard
  4. Repeat for an Organizer account (organizer@test.com)

Step 2: Test Event Creation (Organizer)

  1. Login as Organizer (organizer@test.com)
  2. Click "Create Event" button
  3. Fill in event details:
    • Title: Tech Conference 2026
    • Description: Annual technology conference
    • Date: Select a future date
    • Time: 10:00 AM
    • Venue: Convention Center
    • Capacity: 100
  4. Click "Create Event"
  5. Verify event appears in dashboard

Step 3: Test Event Registration (Participant)

  1. Logout and login as Participant (participant@test.com)
  2. Go to "Events" page
  3. Find the event you created
  4. Click "Register Now"
  5. Confirm registration
  6. Verify success message appears

Step 4: Test Digital Ticketing

  1. Go to "My Tickets" page
  2. Verify your ticket is displayed with:
    • Event details
    • Unique Ticket ID
    • QR Code (this is your digital ticket!)
  3. Take a screenshot of the QR code for testing

Step 5: Test QR Code Scanning (Organizer)

  1. Logout and login as Organizer
  2. Go to Dashboard → Click "Scan QR" for your event
  3. Click "Start Scanner"
  4. Allow camera access when prompted
  5. Show the QR code from Step 4 to the camera
  6. Verify attendance is marked successfully
  7. Try scanning the same QR again - should show "Already attended"

Step 6: Test Email Notifications

⚠️ Email Setup Required: To test email notifications, you need to configure email credentials in backend/.env

Update these values in backend/.env:

EMAIL_HOST=smtp.gmail.com
EMAIL_PORT=587
EMAIL_USER=your_email@gmail.com
EMAIL_PASS=your_app_password

For Gmail, you need to generate an App Password (requires 2FA enabled)

After configuration, register for an event and check your email for the confirmation with QR code!

Step 7: Test Reports & Analytics

  1. Login as Organizer
  2. Go to Dashboard
  3. Click "View Report" for your event
  4. Verify you see:
    • Total registrations
    • Attendance count
    • Attendance rate
    • List of participants with attendance status

API Testing (Optional)

Test API Endpoints Directly

You can test the API using tools like Postman or curl:

# Health Check
curl http://localhost:5000/api/health

# Register User
curl -X POST http://localhost:5000/api/auth/register \
-H "Content-Type: application/json" \
-d '{"name":"Test User","email":"test@test.com","password":"test123","role":"participant"}'

# Get Events
curl http://localhost:5000/api/events

Troubleshooting

Common Issues

  • CORS Error: Make sure backend is running on port 5000
  • MongoDB Connection Error: Ensure MongoDB service is running
  • Camera Access Denied: Grant camera permissions for QR scanning
  • Email Not Sending: Check email credentials in .env file
